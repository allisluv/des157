Alli's DES 157 Project Needfinding

A portfolio building website for kids...

  1. Needs to load quickly.
  2. Needs to be clear where links will take the user.
  3. Needs a clear hierarcy based on what the user is most likely looking for.
  4. Needs clear indications of what is clickable and what is not.
  5. Needs clear indications of which links leave the page and which do not.
  6. Needs strong visual language to go with written descriptions and instructions.
  7. Needs to ask only one thing of the user at a time.
  8. Needs to allow enough options to create individuality.
  9. Needs to limit options to make decision making easier.
  10. Needs to give both positive and negative feedback.
